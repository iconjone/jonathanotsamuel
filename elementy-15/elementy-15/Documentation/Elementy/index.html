Getting Started

Unpack the archive and find out the folder "HTML". There are all the template files in this folder.

Upload the template files (inside the folder "HTML") to the server. For the correct working of all functions of the template, especially JavaScript plugins, you must upload the template to web-server or local server

Due to local security restrictions, some things will not work locally, especially in Chrome. But everything will work once the files have been uploaded to your web-server or local server.

Newsletter

If you want to use the newsletter you need to have a MailChimp account (the funcional Newsletter works only with MailChimp - http://mailchimp.com/)

First we need to access the MailChimp dashboard to gather their HTML code. Sign into your account or create a new free account. Click “Lists” from the main menu and you’ll get a table of all your subscription lists. You might want to use an existing list, or create a new list for this tutorial. Either way click on the list’s name and we get another table organizing all the subscribers by e-mail address.

Any new list will start out empty but it doesn’t matter how many subscribers you have. On this page click “Signup forms” and we’ll be presented with some options. General forms are too basic and we don’t have control over the code. Select “embedded forms” and you’ll get a sleek interface to copy/paste the HTML. Chosen Super Slim over Classic.

You need action attribute. Copy the action URL attribute of MailChimp form. Everything in the URL is identical up until post-json, which was originally post in the MC template. The post action returns a typical completion page we’re all fElementyr with. post-json will return a JSON response based on the success or failure of the e-mail subscription. Please note I’ve also appended &amp;c=? to the end of this URL.

Insert the resulting url in the form of your website

For more information read - http://designshack.net/articles/css/custom-mailchimp-email-signup-form/

Twitter

To Use twitter feeds you will need a small part of HTML and JavaScript code.Twitter feeds uses a a div with id "twitter-feeds"

<div id="twitter-feeds"></div>

But the real magic happens in JavaScript. In document you will found code that makes twitter feeds work.

//TWITTER FEED----------------------------------------------------
 
$(document).ready(function() {
 
    $("#twitter-feeds").tweet({
 
         // join_text: false,
 
          username: "abcgomel", // Change username here
 
          modpath: './js/twitter/', // Twitter files path
 
          avatar_size: false, // you can active avatar
 
          count: 3, // number of tweets
 
          loading_text: "loading tweets..."
 
    });
 
});

You'll have to replace username "abcgomel" with your own, and you can change the "count:3" wich represents the number of tweets.

Then, you need a consumer key and secret keys. Get one from dev.twitter.com/apps.Edit twitter/index.php file and replace variables with your Consumer and oAuth Keys.

 // Your Twitter App Consumer Key
 
    private $consumer_key    = 'YOUR_CONSUMER_KEY';
 
 
 
    // Your Twitter App Consumer Secret
 
    private $consumer_secret = 'YOUR_CONSUMER_SECRET';
 
 
 
    // Your Twitter App Access Token
 
    private $user_token      = 'YOUR_ACCESS_TOKEN';
 
 
 
    // Your Twitter App Access Token Secret
 
    private $user_secret     = 'YOUR_ACCESS_TOKEN_SECRET';

Please, don't forget to connect the necessary Javascript plugin for twitter feed 

<!-- JQUERY TWEETS -->
<script src="js/twitter/jquery.tweet.js"></script>

  <!-- LOADER -->	
  <div id="loader-overflow">

  </div>

Necessary JS code is already in main.js

  /* --------------------------------------------
    PAGE LOADER
  --------------------------------------------- */
  $("body").imagesLoaded(function(){
    $(".loader-cont").fadeOut();
    $("#loader-overflow").delay(200).fadeOut(700);
  });
